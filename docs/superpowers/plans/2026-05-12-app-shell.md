# App Shell (Phase B) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Port the full visual surface of `sample_screens/feed.html` (header, left/right sidebars, center stub column) into the authenticated `_app` layout, with a real theme toggle and logout wired up; everything API-bound (search, notifications, suggestions, friends, events, stories, composer, feed) renders sample data and fires `toast.info("…coming soon")` on interaction.

**Architecture:** Presentational components in `src/components/shell/`; four new shadcn primitives in `src/components/ui/`; theme as a small Zustand store with `localStorage` persistence and a no-flash inline script; one centralized `src/data/sample-shell.ts` for every static placeholder array; inline SVG icons ported verbatim from the sample into `src/components/shell/icons/`. The current `_app` route's minimal header is replaced wholesale by `<AppLayout />`, which composes header + floating theme toggle + 3-column grid (`hidden lg:block` sidebars, full-width center on `<lg`).

**Tech Stack:** React 19, TanStack Router (file-based), Tailwind v4 (CSS-only tokens), shadcn/ui (new-york, slate base) — adds `dropdown-menu`, `avatar`, `scroll-area`, `separator`. Zustand for theme state. sonner for toasts. Radix peers via shadcn.

**Spec:** `docs/superpowers/specs/2026-05-12-app-shell-design.md`

**Testing note (deviation from default writing-plans):** Per the approved spec (§Testing Strategy), Phase B is gated by a manual smoke checklist instead of unit tests — the surface is overwhelmingly static composition plus one stateful slice (theme), and the spec explicitly opts out of automated tests. A final verification task runs the §Testing Strategy checklist before declaring done.

- [ ] **Step 1: Write `src/lib/theme.ts`**

```typescript
export type Theme = "light" | "dark"

export const THEME_STORAGE_KEY = "theme"

export function readStoredTheme(): Theme {
  try {
    const value = window.localStorage.getItem(THEME_STORAGE_KEY)
    return value === "dark" ? "dark" : "light"
  } catch {
    return "light"
  }
}

export function applyTheme(theme: Theme): void {
  const root = document.documentElement
  if (theme === "dark") {
    root.classList.add("dark")
  } else {
    root.classList.remove("dark")
  }
}
```

```typescript
import { create } from "zustand"
import {
  type Theme,
  THEME_STORAGE_KEY,
  applyTheme,
  readStoredTheme,
} from "@/lib/theme"

type ThemeState = {
  theme: Theme
  setTheme: (theme: Theme) => void
  toggle: () => void
}

function persistTheme(theme: Theme): void {
  try {
    window.localStorage.setItem(THEME_STORAGE_KEY, theme)
  } catch {
    // best-effort; private mode / quota — swallow.
  }
}

const initialTheme: Theme =
  typeof window === "undefined" ? "light" : readStoredTheme()

export const useThemeStore = create<ThemeState>((set, get) => ({
  theme: initialTheme,
  setTheme: (theme) => {
    applyTheme(theme)
    persistTheme(theme)
    set({ theme })
  },
  toggle: () => {
    const next: Theme = get().theme === "dark" ? "light" : "dark"
    applyTheme(next)
    persistTheme(next)
    set({ theme: next })
  },
}))
```
