# Frontend Infrastructure Setup — Design

**Date:** 2026-05-12
**Status:** Approved (awaiting user review of written spec)
**Reference project:** `/home/shaikh/my_projects/hono_project_payroll_saas/hr-payroll-saas-fe`

## Purpose

Establish the infrastructure layer of the social-feed frontend: API client, Kubb code generation, TanStack Query, auth flow, route shells, and base + authenticated layouts. **No feature UI is built in this phase.** The output is a project that boots, bootstraps auth via refresh-cookie + `/me`, routes the user to either `/auth/login` or `/` based on auth state, and has Kubb-generated typed clients ready to consume.

## Non-Goals

- Login or signup forms (TanStack Form integration deferred)
- Feed list rendering (TanStack Virtual deferred)
- Any real API calls beyond `POST /auth/refresh`, `POST /auth/logout`, `GET /auth/me`
- Theme provider, error boundary, modal host, splash-screen UI polish
- Multi-device "logout all" flow
- Permissions / role-based access
- Tests for new modules (set up later when there is logic worth testing)

## Architectural Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Framework | Vite SPA | User chose to strip TanStack Start; matches reference project 1:1 |
| Proxy | One combined entry: `/api/*` → `localhost:8787` with `/api` stripped | User requirement: one proxy, not three |
| API spec | Single combined OpenAPI doc at `:8787/docs` → `src/gen/api/` | User chose single spec over per-module specs |
| Auth model | Hybrid: access token in memory, refresh token in HttpOnly cookie | User chose C; lower CSRF risk, supports future non-browser clients |
| Auth state | Minimal Zustand store | User chose Zustand over TanStack-Query-only model |
| HTTP | `ky` with `credentials: "include"` | Refresh cookie auto-attached; matches reference |
| Forms | `@tanstack/react-form` (installed, not used yet) | Deferred to next phase |
| Lists | `@tanstack/react-virtual` (installed, not used yet) | Deferred to next phase |
| Validation | `zod` v4, schemas generated by `@kubb/plugin-zod` | Strict validation, no hand-written schemas |
| Toaster | `sonner` | Matches reference |

## Components & Responsibilities

Each module has one clear job, communicates through narrow interfaces, and can be reasoned about in isolation.

### `src/lib/env/index.ts`
**Purpose:** Validate `import.meta.env` at module load. Fail-fast on misconfiguration.
**Exports:** `env` (validated runtime config), `EnvSchema`, `parseEnv`.
**Inputs:** `import.meta.env`.
**Outputs:** `{ apiUrl, appName, isDev, isProd }`.

### `src/lib/api-error.ts`
**Purpose:** Define the `ApiError` class and the Kubb-compatible `RequestConfig` / `ResponseConfig` / `Client` types.
**Exports:** `ApiError`, `RequestConfig`, `ResponseConfig`, `ResponseErrorConfig`, `Client`.
**Depends on:** `ky` types only.

### `src/lib/api-client.ts`
**Purpose:** The one ky instance used by all generated clients. Attaches `Authorization` header, runs single-flight refresh on 401, surfaces connectivity toasts.
**Exports:** `api` (KyInstance), `notifyConnectivityIssueIfNeeded`.
**Depends on:** `auth.ts` (for token + refresh), `env.ts`, `sonner`.

### `src/lib/kubb-clients/create-client.ts`
**Purpose:** Adapter that wraps a `KyInstance` into the function shape Kubb's generated code calls. Translates ky's `HTTPError` into our `ApiError`.
**Exports:** `createKubbClient(ky) → Client`, plus re-exported types.
**Depends on:** `ky`, `api-error.ts`.

### `src/lib/kubb-clients/api-client.ts`
**Purpose:** Default-export Client backed by the *auth-hooked* `api` instance. This is what `kubb.config.ts`'s `importPath` points to — every generated function receives this client by default.

### `src/lib/kubb-clients/bare-api-client.ts`
**Purpose:** Default-export Client backed by a *plain* ky instance with no auth hooks. Used by `auth.ts` for `/auth/refresh` and `/auth/logout` to avoid the 401-retry recursion. Used by `auth-init.ts` for `/auth/me` during bootstrap (because the in-memory token is still being established).
**Why two clients exist:** the hooked client would loop forever if the refresh endpoint itself returned 401. The bare client breaks the cycle.

### `src/lib/auth.ts`
**Purpose:** Owns the in-memory access token, the single-flight refresh promise, the BroadcastChannel for cross-tab logout, and the logout helper.
**Exports:** `getAccessToken`, `setAccessToken`, `clearAuth`, `tryRefreshToken`, `broadcastLogout`, `onAuthMessage`, `logoutCurrentDevice`.
**Depends on:** `bare-api-client`, generated `refreshToken` + `logout` client functions, `use-auth.ts` (lazy-imported to avoid circular dep).

### `src/lib/auth-init.ts`
**Purpose:** App-boot bootstrap. Refreshes once, fetches `/auth/me`, populates Zustand. Idempotent (`isInitialized` guard).
**Exports:** `initializeAuth(): Promise<void>`.
**Called from:** `__root.tsx`'s `beforeLoad`.

### `src/lib/auth-redirect.ts`
**Purpose:** Safe redirect-after-login helper. Prevents open-redirect attacks and prevents redirect loops back to auth routes.
**Exports:** `safeRedirectPath(value): string | null`, `buildLoginRedirectSearch(currentPath): { redirect?: string }`.
**Auth route set:** `["/auth/login", "/auth/signup"]`.

### `src/lib/query-client.ts`
**Purpose:** One `QueryClient` with standard defaults (5min stale, 10min gc, no retry on 401/403/404).
**Exports:** `queryClient`.

### `src/hooks/use-auth.ts`
**Purpose:** Minimal Zustand store: `{ user, isAuthenticated, isInitialized }` + setters + reset.
**Note:** The inline `User` type is a placeholder until the first kubb generation produces the real shape, which can then be imported from `@/gen/api/types`.

### `src/routes/__root.tsx`
**Purpose:** Root route. `beforeLoad: initializeAuth`. Mounts the cross-tab logout listener.

### `src/routes/auth.tsx`
**Purpose:** Layout route for `/auth/*`. If already authenticated, redirects away (honors `?redirect=` param).
**Children:** `auth/login.tsx`, `auth/signup.tsx` (empty shells in this phase).

### `src/routes/_app.tsx`
**Purpose:** Pathless layout group for all authenticated routes. Guards with `beforeLoad` that redirects to `/auth/login` (with `redirect` param) if not authenticated. Renders top nav with logout button.
**Children:** `_app/index.tsx` ("Feed coming soon" placeholder at URL `/`).

### `src/main.tsx`
**Purpose:** App entry. Wires `RouterProvider` + `QueryClientProvider` + `Toaster` + `ReactQueryDevtools`. Replaces the TanStack Start entry.

### `kubb.config.ts`
**Purpose:** Single-API Kubb configuration. Generates types, zod schemas, clients, and react-query hooks from `./specs/api.json` into `./src/gen/api/{types,zod,clients,hooks}`.

### `scripts/fetch-specs.sh`
**Purpose:** Curl `http://localhost:8787/docs` → `./specs/api.json`. Wired into `pnpm api:generate`.

## Data Flow

### Cold boot (hard reload to `/`)
1. Browser loads `index.html` → `main.tsx` mounts React tree.
2. `RouterProvider` starts processing the root route.
3. `__root.tsx` `beforeLoad` → `initializeAuth()`.
4. `tryRefreshToken()` → bare client calls `POST /auth/refresh`. Browser auto-attaches HttpOnly refresh cookie. Backend responds with `{ accessToken }`. Token saved to module-level `let`.
5. `getCurrentUser({ client: bareApiClient, headers: { Authorization: Bearer ... } })` → backend returns user. Zustand store populated.
6. `isInitialized = true`. Bootstrap returns.
7. `_app.tsx` `beforeLoad` → reads Zustand → authenticated → passes.
8. `_app/index.tsx` renders ("Feed coming soon").

### Cold boot (unauthenticated)
Same up to step 4. Refresh fails → catch block sets `isInitialized = true` with empty user. `_app.tsx` `beforeLoad` → not authenticated → throws `redirect({ to: "/auth/login", search: { redirect: "/" } })`. `auth.tsx` renders, login shell appears.

### 401 mid-session (token expired)
1. Component calls a generated client function (e.g. `getFeed({})`).
2. Generated function uses `api-client.ts` (auth-hooked).
3. ky's `beforeRequest` attaches `Authorization: Bearer <stale-token>`.
4. Backend returns 401.
5. `afterResponse` hook fires `tryRefreshToken()`. Single-flight: concurrent 401s share the same promise.
6. On success: hook retries the original request with the new token via `ky.retry()`. Caller sees no error.
7. On failure: `clearAuth()`, toast, hard redirect to `/auth/login`.

### Cross-tab logout
Tab A calls `logoutCurrentDevice()` → posts `{ type: "logout" }` to the `"auth"` BroadcastChannel. Tab B's `__root.tsx` listener sees the message → clears local auth → redirects to `/auth/login`.

## Error Handling

| Layer | Strategy |
|---|---|
| Network / timeout in `api-client` | `notifyConnectivityIssueIfNeeded` toast |
| HTTP error in `create-client` | Wrapped into `ApiError` with status, body, message extracted from `body.message` or `body.error` |
| 401 in `api-client` | Single-flight refresh + retry; on failure, hard redirect |
| Refresh failure in `auth-init` | Caught, store reset, `isInitialized = true` — app continues as anonymous |
| Bad env config | Throws at module load with concatenated zod issues |
| `logoutCurrentDevice` network failure | Local state cleared anyway (then redirect) |

## Testing Strategy

No automated tests added in this phase. Manual smoke verification at end of execution:

- `pnpm typecheck` → 0 errors.
- `pnpm dev` → app loads at `http://localhost:3000`.
- Hard reload at `/` while logged out → lands on `/auth/login`. Network panel shows `POST /api/auth/refresh` → 401 (or whatever the backend returns when no cookie), no console errors.
- Hard reload at `/` while logged in (cookie present) → stays at `/`. Network panel shows `POST /api/auth/refresh` → 200, then `GET /api/auth/me` → 200.
- Click logout in top nav → redirected to `/auth/login`. `Application > Cookies` shows refresh cookie cleared by backend.
- Open `/` in a second tab while logged in, then logout from tab 1 → tab 2 redirects to `/auth/login` shortly after.

Test suite scaffolding (Vitest is already in `devDependencies`) stays in place but no tests are written until there is logic with branching worth testing.

1. **Tear down TanStack Start.** Remove `@tanstack/react-start`, `@tanstack/react-router-ssr-query`, `@tanstack/devtools-vite` from `package.json`. 4. **Write `vite.config.ts`** (Vite SPA, combined `/api` proxy with rewrite).
5. **Update `tsconfig.json`** to map `@/*` → `./src/*`.
6. **Create `src/lib/env/index.ts`, `src/lib/api-error.ts`, `src/lib/query-client.ts`.**
7. **Create `src/lib/kubb-clients/create-client.ts`, `api-client.ts`, `bare-api-client.ts`.**
8. **Create `kubb.config.ts` and `scripts/fetch-specs.sh`.** `chmod +x` the script.
9. **Run `pnpm api:generate`.** Verifies backend reachability and spec validity. Produces `src/gen/api/`.
10. **Inspect generated client function names.** Confirm `refreshToken`, `logout`, and the `/auth/me` getter exist with the expected names. Adjust imports in step 11 if the names differ from assumptions.

12. **Create routes:** `__root.tsx`, `auth.tsx`, `auth/login.tsx`, `auth/signup.tsx`, `_app.tsx`, `_app/index.tsx`.
13. **Create `src/main.tsx`.**
14. **`pnpm typecheck`** → must be 0 errors before declaring done.
15. **`pnpm dev`** → run the smoke verification listed under Testing Strategy.

## Risks

- **Kubb generation failure on first run (step 9).** Possible if the backend OpenAPI spec has unsupported constructs (recursive refs, missing `operationId`, polymorphic responses). Mitigation: `pluginOas({ validate: false })` is already in the config; if generation fails we either fix the spec on the backend or selectively exclude problem paths.
- **TanStack Start tear-down may leave orphan dependencies.** Mitigation: after step 1, run `pnpm install` and grep for any `@tanstack/react-start` imports.
- **Refresh endpoint shape mismatch.** If the backend returns `{ token }` instead of `{ accessToken }`, step 10 catches it; adjust `auth.ts`.
